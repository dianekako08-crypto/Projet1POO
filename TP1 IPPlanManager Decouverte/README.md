TP1 - IPPlan-Manager## Objectif du TP Ce TP permet de découvrir les premières classes Java du projet IPPlanManager. L'objectif est de s'initier à la Programmation Orientée Objet (POO) en modélisant des entités réseau simples.## Classes crééesAdresseIP : Gestion des adresses IPv4.ReseauIP : Définition des sous-réseaux.InterfaceReseau : Modélisation d'une carte réseau.Equipement : Représentation d'un matériel informatique (PC, Routeur, etc.).Main : Classe principale pour l'exécution et les tests.## Travail réalisé Dans ce TP, j'ai implémenté la structure de base du logiciel. J'ai créé plusieurs objets pour simuler un parc informatique :Instanciation d'objets AdresseIP (ex: 192.168.1.1).Configuration d'InterfacesReseau liant une adresse IP à une interface physique.Création d'Equipements complets possédant un nom et une interface.Tests effectués : Affichage dans la console des caractéristiques des équipements pour vérifier la bonne transmission des données entre les objets.## Réponses aux questions1. Pourquoi une adresse IP a-t-elle été représentée par une classe au lieu d’être simplement stockée dans une variable String partout dans le programme ?Une adresse IP n'est pas qu'une simple chaîne de caractères ; c'est un concept avec des règles métier (4 octets entre 0 et 255). Utiliser une classe permet de regrouper la donnée et les méthodes de validation ou de calcul (comme la conversion binaire ou le calcul de masque) au même endroit, rendant le programme plus robuste et évolutif.2.Quelle est la différence entre une classe et un objet ?La classe est le modèle, la définition ou le "moule" (ex: la classe AdresseIP). L'objet (ou instance) est la concrétisation en mémoire de ce modèle avec des valeurs spécifiques (ex: l'adresse 192.168.1.1).3. Quel est le rôle du constructeur dans une classe Java ?Le constructeur est une méthode spéciale appelée lors de la création d'un objet avec le mot-clé new. Son rôle principal est d'initialiser les attributs de l'objet pour s'assurer qu'il est dans un état valide dès sa naissance.4. Pourquoi la classe InterfaceReseau contient-elle un objet de type AdresseIP ?C'est ce qu'on appelle la composition (relation "possède un"). Une interface réseau physique possède nécessairement une adresse IP pour communiquer. En programmation, on traduit cela en plaçant un objet AdresseIP comme attribut de la classe InterfaceReseau.5. Pourquoi la classe Equipement contient-elle un objet de type InterfaceReseau ?De la même manière, un équipement (ordinateur, routeur) est composé de composants matériels. L'interface réseau est l'un de ces composants. Cette structure reflète la réalité physique du matériel informatique.6.Quelle est la limite actuelle de la classe Equipement dans ce TP ?La limite principale est qu'un équipement ne peut posséder qu'une seule interface réseau. Dans la réalité, de nombreux appareils (comme les serveurs ou les routeurs) sont "multi-homed", c'est-à-dire qu'ils possèdent plusieurs interfaces (Ethernet et Wi-Fi par exemple).7.Pourquoi cette première version n’est-elle pas encore suffisante pour produire automatiquement un plan d’adressage IP ?Cette version ne fait que stocker et afficher des informations. Elle ne possède pas encore l'intelligence algorithmique pour calculer les plages d'adresses, identifier l'adresse réseau/broadcast, ou détecter automatiquement les conflits d'adresses au sein d'un parc informatique.
